# StyleSlice 一分钟用户链路 HTML 演示设计

## 目标

制作一个可在浏览器中直接打开的 16:9 横屏 HTML 演示，用 45–55 秒展示 StyleSlice 的完整用户使用链路。演示只回答“用户如何使用、最终得到什么”，不加入市场规模、技术架构、商业模式或后续规划。

传播任务：演示结束时，观众应理解 StyleSlice 能把一张用户说不清风格的截图，转化为可理解、可复用、可积累的个人审美资产。

## 页面结构

演示共 4 页，每页只承担一个叙事任务，并使用一张真实产品界面作为主视觉。

1. **上传截图｜把喜欢交给 StyleSlice**
   - 使用界面图 `1.png`。
   - 观众理解：用户从手机中选择一张喜欢但无法准确描述风格的截图。
   - 屏幕文案：`把喜欢的截图，交给 StyleSlice。`

2. **AI 拆解｜看懂风格，也能再次生成**
   - 使用界面图 `2.png`。
   - 观众理解：系统提取色彩、构图和设计关键词，并生成可复制的 AI 生图提示词。
   - 屏幕文案：`AI 拆出色彩、构图与风格语言。`

3. **收藏复用｜每一次喜欢都有去处**
   - 使用界面图 `3.png`。
   - 观众理解：分析结果进入个人切片库，之后可以检索、打开和复用。
   - 屏幕文案：`收藏成切片，让灵感随时能被找回。`

4. **审美沉淀｜收藏最终变成“我是谁”**
   - 使用审美画像界面图 `codex-clipboard-9fd224da-6ee6-4fd4-8fc4-2e457d471da6.png`。
   - 观众理解：多次收藏汇总为审美画像，呈现稳定偏好与同频设计师。
   - 屏幕文案：`积累的不是图片，而是你的审美画像。`

## 视觉方向

- 延续产品现有黑、白、绿色体系，绿色只作为进度与强调色。
- 每页使用一张手机界面作为最大视觉主体；不制作复杂信息卡片或技术图。
- 页面包含固定的 StyleSlice 标识、步骤编号 `01–04`、一句主文案和短说明。
- 四页构图有节奏变化，但保持统一的边距、字体、线条和页面标记。
- 动效以淡入、轻微位移和图片缩放为主，避免炫技或影响讲解。

## 交互与播放

- 左右方向键、空格键、点击左右区域均可翻页。
- 支持触屏左右滑动。
- 支持全屏按钮与浏览器原生全屏。
- 页面显示 4 段进度条和当前页码。
- 默认不自动翻页，避免演讲节奏被网络或现场停顿打断。
- HTML 使用本地资源，不依赖网络字体、CDN 或第三方脚本。

## 文件与技术结构

- 最终交付为一个演示目录，包含 `index.html` 和 `assets/` 图片目录。
- CSS 与 JavaScript 内联在 `index.html` 中，便于拷贝和离线展示。
- 图片复制到演示目录，使用稳定的英文文件名，避免依赖桌面临时路径。
- 画布按 16:9 缩放，适配电脑投影；小窗口下保持完整画面并留黑边。

## 验收标准

- 四张界面图均清晰显示且顺序正确。
- 讲解按每页约 10–13 秒可在 1 分钟内完成。
- 1366×768、1440×900、1920×1080 下无文字截断、元素溢出或非预期重叠。
- 键盘、鼠标、触摸翻页与全屏功能可用。
- 断网后仍能正常打开、显示和播放。
- 演示中不出现技术架构、市场规模、商业模式或未实现能力的表述。
