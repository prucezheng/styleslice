# 黑客松四人分工

## 项目目标

用户上传一张或多张设计参考图片，系统生成：

1. 通用视觉语言规范 `.md`
2. 色卡＋视觉令牌 Style Card（排除 Typography）

## 四人分工

| 成员 | 职责 | 交付 |
| --- | --- | --- |
| 1. 产品＋UI | 页面流程、Figma、Style Card 模板、演示 | 完整设计稿和演示流程 |
| 2. 前端 | 上传、分析、结果、资料库页面 | 可操作的前端页面 |
| 3. 后端 | 图片接收、任务处理、结果保存与读取 | 上传、分析、资料库接口 |
| 4. AI | 图片视觉分析、Prompt、JSON、MD 生成 | 稳定的结构化分析结果 |

## 统一开发约定

- AI 先输出统一 JSON，再生成 MD 和 Style Card。
- 后端未完成前，前端先用固定 JSON 开发。
- 每个人先完成主流程，不做账号、团队协作等次要功能。
- 准备一套缓存结果，避免比赛现场调用失败。

## 最终演示流程

上传图片 → AI 解析 → 生成 MD 和 Style Card → 保存到资料库 → 在新设计任务中调用 MD
